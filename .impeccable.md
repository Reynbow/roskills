## Design Context

### Users

- Primary audience appears to be **Ragnarok Online (pre-renewal) players** using a web tool to plan class skill builds and research card drops/spawns.
- Likely used **during theorycrafting** (desktop) and **in-the-moment reference** (mobile), with sharing links as a supporting workflow.

### Brand Personality

- Voice: **practical, game-native, no-fluff**
- Feel: **confident, focused, a little nostalgic** (RO-adjacent)

### Aesthetic Direction

- Theme: **dark-first** (current UI is dark with cool-blue accents)
- Visual language: **RO-inspired pixel UI** (self-hosted `Galmuri11`), crisp edges, compact information density, “tool not landing page”.
- Accents: restrained, functional highlights for state and affordance (filters, pips, toasts), avoid decorative neon/glass.

### Design Principles

1. **Game-native clarity**: prioritize RO-familiar patterns, terminology, and compact density over marketing-style layouts.
2. **Fast scanning > flourish**: hierarchy, spacing rhythm, and consistent alignment beat extra decoration.
3. **Make state obvious**: selection, copied feedback, prerequisites, and filters should read instantly.
4. **Mobile is not a shrink**: adapt into list-first, thumb-friendly controls with progressive disclosure.
5. **Delight only when functional**: animation and effects must communicate feedback (copy, hover, focus), and respect reduced-motion.
